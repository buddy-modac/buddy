Design Direction

핵심 기능 하나는 뾰족하게, 경험은 친근하게

1

확실한 핵심 기능

캡처한 화면을 AI가 이해하고 바로 응답하는 흐름을 MVP의 중심으로 둡니다.

2

적당한 비주얼 완성도

해커톤 기간 안에 구현 가능하면서도 보는 즉시 제품 콘셉트가 이해되도록 만듭니다.

3

취향을 존중하는 버디

MBTI 페르소나와 캐릭터 커스터마이징으로 도구보다 동료에 가까운 느낌을 줍니다.

회의록의 “확실한 핵심 기능 + 적당한 비주얼 완성도”를 발표의 제품 철학으로 정리.

Solution Concept

Buddy는 화면 위에 상주하는 AI 작업 보조 도구입니다

상주형 캐릭터

화면 한쪽에 머무르며 사용자의 흐름을 방해하지 않고 필요한 순간에 반응합니다.

Capture → Understand → Reply

캡처 기반 AI

선택한 화면을 요약, 번역, 설명하거나 이미지에 대한 질문에 답합니다.

화면을 선택하면 AI가 이해하고, 데스크톱 캐릭터가 말풍선으로 도와주는 경험.

Q&A Prep

예상 질문

왜 캐릭터가 필요한가요?

기능만 제공하는 도구보다 현재 화면에 머무르는 버디가 더 낮은 진입 장벽과 친근한 상호작용을 만듭니다.

기존 캡처/AI 도구와 무엇이 다른가요?

캡처와 AI 응답을 한 화면 안에서 이어 붙이고, 페르소나와 캐릭터를 통해 반복 사용 경험을 강화합니다.

개인정보와 API Key는 어떻게 다루나요?

Claude/OpenAI 키는 Unity가 아니라 Mac mini 서버 환경변수로 관리하고, 사용자가 명시적으로 캡처한 화면만 전송합니다.

왜 Unity를 검토했나요?

데스크톱 캐릭터와 비주얼, 에셋 활용에 강점이 있고 짧은 시간 안에 시각적 프로토타입을 만들기 좋습니다.

실패하거나 어려웠던 점은?

아이디어가 넓어질수록 MVP가 흐려졌고, 캡처 기반 AI라는 핵심 플로우로 다시 좁히는 과정이 필요했습니다.

앞으로 가장 먼저 개선할 부분은?

시연 안정성, OCR/이미지 이해 품질, 캡처 후 액션 선택 UX를 우선 개선합니다.
